介绍

Anaconda 是一个免费开源的 Python 和 R 的发行版本，主要用于解决机器学习和数据科学应用中的环境管理和部署难题。在对接量化数据做分析的时候用到，用起来还蛮方便的，记录一下。

miniconda 是 Anaconda 的最小化干净安装版本，仅包含基础的 conda、Python、基础的依赖包和 zlib 等常用包。喜欢服务器干净整洁的同学可以选择 miniconda。
Anaconda 中的 conda 则是一个开源跨平台的包管理工具和系统管理工具。关于 conda 包管理和系统管理得分开理解。conda 作为包管理工具时一如 pip 之于 Python，用来管理环境包的安装等，在 Anaconda 中的 pip 的作用可以视作 conda 的子集。而作为系统管理工具时又有点像 docker 支持快速构建独立的虚拟环境都体现了拿来即用的思想，同时它们都是进程级别的隔离。不过中间还是有点区别的--它们的区别可以理解为虚拟环境和容器的区别：conda 和 venv、virtualenv 一样通过环境变量 $PATH 等替换实现虚拟环境隔离，所以虚拟环境安装的工具（如 python3.8）宿主机也能找到其解释器，所以虚拟环境中的程序在不激活环境时也可以通过拼装对应解释器的完整路径调用运行。而 docker 的容器环境隔离则是依托 linux 系统的 namespace 技术实现隔离，容器安装的特定版本语言解释器在宿主机是没有的。关于虚拟环境、容器和虚拟机环境隔离的再抽空写专门的总结吧。

安装

首先到 anaconda 官网找到自己想下载的版本，wget 下载并执行之

[root@iZwz9f3z1uc58n1ogxlk8aZ ~]# wget https://mirrors.tuna.tsinghua.edu.cn/anaconda/archive/Anaconda3-5.3.1-Linux-x86_64.sh
[root@iZwz9f3z1uc58n1ogxlk8aZ ~]# bash Anaconda3-5.3.1-Linux-x86_64.sh -u

Welcome to Anaconda3 5.3.1

In order to continue the installation process, please review the license
agreement.
Please, press ENTER to continue

[no] >>>

>>> yes

然后是指定安装路径，懒得换就直接 enter

/root/anaconda3

  - Press CTRL-C to abort the installation

踩坑记录

No module named conda

最后记录一下踩的坑
修改环境的 python 版本时， 有一次执行

conda install python=3.8

后得到了奇怪的报错：ModuleNotFoundError: No module named 'conda' 可有点秀,命令自己报错说没有自己的模块吗哈哈

Preparing transaction: done
Verifying transaction: done
Executing transaction: done
Traceback (most recent call last):
  File "/root/anaconda3/bin/conda", line 7, in <module>
    from conda.cli import main
ModuleNotFoundError: No module named 'conda'
[root@iZwz9f3z1uc58n1ogxlk8aZ ~]#

这个问题网上很多答案也不难查，简单说就是修改 base 环境的 python 改变导致解释器找不到当前conda 包。所以不要随便修改 base 环境 python 版本，可以多开个虚拟环境嘛
解决的话只要找回之前安装 Anaconda 的脚本执行

bash Anaconda3-5.1.0-Linux-x86_64.sh -u

即可
附，想要更新 base 环境的 python 版本的话可以参考升级anaconda默认的python版本
